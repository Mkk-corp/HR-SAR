# وثيقة متطلبات الأعمال (BRD)
## نظام إدارة الموارد البشرية — HR Management System

---

**الإصدار:** 1.0
**التاريخ:** 2026-03-28
**الحالة:** قيد التطوير

---

## 1. نظرة عامة على المشروع

### 1.1 الهدف من المشروع

بناء نظام ويب متكامل لإدارة الموارد البشرية يتيح للمؤسسات تسجيل وإدارة بيانات موظفيها ومنشآتها في مكان واحد، مع دعم كامل للغة العربية واتجاه النص من اليمين لليسار (RTL).

### 1.2 نطاق المشروع

يشمل النظام الوحدات التالية:
- **لوحة التحكم** — عرض إحصائيات وملخصات فورية
- **إدارة الموظفين** — تسجيل وتعديل وحذف بيانات الموظفين
- **إدارة المنشآت** — إدارة المنشآت الأساسية والفرعية
- **التقارير** — تقارير الرواتب وتوزيع الموظفين

---

## 2. أصحاب المصلحة

| الجهة | الدور |
|---|---|
| مدير النظام | المستخدم الرئيسي — يملك صلاحيات كاملة على جميع البيانات |
| إدارة الموارد البشرية | إدخال بيانات الموظفين ومتابعة حالاتهم |
| الإدارة المالية | الاطلاع على تقارير الرواتب |

---

## 3. متطلبات الأعمال

### 3.1 وحدة لوحة التحكم

| الرقم | المتطلب |
|---|---|
| BR-D-01 | يجب أن يعرض النظام إجمالي عدد الموظفين المسجلين |
| BR-D-02 | يجب أن يعرض النظام عدد الموظفين النشطين |
| BR-D-03 | يجب أن يعرض النظام إجمالي عدد المنشآت |
| BR-D-04 | يجب أن يعرض النظام إجمالي كتلة الرواتب الشهرية بالريال السعودي |
| BR-D-05 | يجب أن يعرض النظام آخر الموظفين المضافين (حتى 6 موظفين) |
| BR-D-06 | يجب أن يعرض النظام توزيع الموظفين حسب المنشأة على شكل أشرطة بيانية |

### 3.2 وحدة إدارة الموظفين

| الرقم | المتطلب |
|---|---|
| BR-E-01 | يجب أن يتمكن المستخدم من إضافة موظف جديد بالبيانات الكاملة |
| BR-E-02 | يجب أن يكون الرقم التوظيفي فريداً لكل موظف |
| BR-E-03 | يجب أن يتمكن المستخدم من تعديل بيانات أي موظف |
| BR-E-04 | يجب أن يتمكن المستخدم من حذف موظف مع تأكيد مسبق |
| BR-E-05 | يجب أن يتمكن المستخدم من البحث عن موظف بالاسم أو الرقم التوظيفي أو الجنسية |
| BR-E-06 | يجب أن يتمكن المستخدم من تصفية الموظفين حسب المنشأة |
| BR-E-07 | يجب أن يتمكن المستخدم من تصفية الموظفين حسب الحالة الوظيفية |
| BR-E-08 | يجب أن تكون حالات الموظف: نشط، غير نشط، خروج نهائي، خروج مؤقت، اجازة |
| BR-E-09 | يجب أن يتمكن المستخدم من النقر على صف الموظف للانتقال إلى صفحة تفاصيل كاملة |
| BR-E-10 | يجب أن تعرض صفحة تفاصيل الموظف جميع بياناته مقسمةً في أقسام واضحة |
| BR-E-11 | يجب أن يتمكن المستخدم من إرفاق صورة شخصية للموظف |

**البيانات المطلوبة لكل موظف:**

*البيانات الشخصية:*
- الاسم الكامل (إلزامي)
- الرقم التوظيفي (إلزامي — فريد)
- الرقم القومي (إلزامي)
- رقم الهوية (إلزامي)
- الجنسية (إلزامي)
- نوع الموظف: سعودي / اجنبي (إلزامي)
- تاريخ انتهاء الهوية
- تاريخ دخول المملكة
- صورة شخصية (اختيارية)

*بيانات العمل:*
- المنشأة (إلزامي — اختيار من قائمة المنشآت المسجلة)
- المرتب الأساسي بالريال السعودي (إلزامي)
- حالة الموظف (إلزامي)
- عنوان العمل

*بيانات البنك والتواصل:*
- اسم البنك
- رقم الآيبان (IBAN)
- كود الدولة
- رقم الهاتف

### 3.3 وحدة إدارة المنشآت

| الرقم | المتطلب |
|---|---|
| BR-F-01 | يجب أن يتمكن المستخدم من إضافة منشأة جديدة |
| BR-F-02 | يجب أن يدعم النظام نوعين من المنشآت: أساسية وفرعية |
| BR-F-03 | يجب أن ترتبط المنشأة الفرعية بمنشأة أساسية واحدة |
| BR-F-04 | يجب أن ترث المنشأة الفرعية جميع بيانات المنشأة الأساسية عدا الاسم عند الإنشاء |
| BR-F-05 | يجب أن تبقى جميع حقول المنشأة الفرعية قابلة للتعديل بعد الإنشاء |
| BR-F-06 | يجب أن يتمكن المستخدم من عرض صفحة تفاصيل لكل منشأة |
| BR-F-07 | يجب أن تعرض صفحة التفاصيل: بيانات المنشأة، قائمة موظفيها، وقائمة المنشآت الفرعية (للأساسية فقط) |
| BR-F-08 | يجب أن يتمكن المستخدم من إضافة منشأة فرعية مباشرة من صفحة تفاصيل المنشأة الأساسية |
| BR-F-09 | يجب أن يتمكن المستخدم من تعديل بيانات المنشأة |
| BR-F-10 | يجب أن يتمكن المستخدم من حذف منشأة مع تأكيد مسبق |
| BR-F-11 | يجب أن تعرض صفحة تفاصيل المنشأة شجرة هرمية مرئية عند وجود علاقة أب/فرع — للمنشأة الأساسية: تُظهر نفسها كجذر ومنشآتها الفرعية كأفرع — للمنشأة الفرعية: تُظهر الأب كجذر وجميع الأخوات مع تمييز المنشأة الحالية — العقد في الشجرة قابلة للنقر للانتقال المباشر |

**البيانات المطلوبة لكل منشأة:**

| الحقل | الوصف | إلزامي |
|---|---|---|
| اسم المنشأة | الاسم الرسمي للمنشأة | نعم |
| نوع المنشأة | أساسية / فرعية | نعم |
| المنشأة الأساسية | مرجع للمنشأة الأم (للفرعية فقط) | مشروط |
| الرقم الوطني الموحد | رقم الهوية الوطنية للمنشأة | لا |
| رقم السجل التجاري | رقم السجل التجاري الرسمي | لا |
| تاريخ إصدار السجل التجاري | تاريخ إصدار السجل | لا |
| الرقم الضريبي | الرقم الضريبي لدى هيئة الزكاة والدخل | لا |
| رقم المنشأة في التأمينات | رقم التسجيل في التأمينات الاجتماعية | لا |
| العنوان الوطني | رابط خريطة أو كود العنوان الوطني | لا |
| موقع العمل | وصف الموقع (حي، مدينة) | لا |

### 3.4 وحدة التقارير

| الرقم | المتطلب |
|---|---|
| BR-R-01 | يجب أن يعرض النظام ملخص الرواتب الإجمالية مصنفةً حسب المنشأة |
| BR-R-02 | يجب أن يعرض النظام توزيع الموظفين حسب الحالة الوظيفية مع النسبة المئوية |
| BR-R-03 | يجب أن يعرض النظام الإجمالي الكلي للرواتب |
| BR-R-04 | يجب أن يعرض النظام الإجمالي الكلي لعدد الموظفين |

## 4. قواعد العمل

| الرقم | القاعدة |
|---|---|
| BRL-01 | لا يمكن لموظفين اثنين امتلاك نفس الرقم التوظيفي |
| BRL-02 | لا يمكن إنشاء منشأة فرعية بدون تحديد منشأة أساسية |
| BRL-03 | لا يمكن تعيين منشأة فرعية كمرجع أب لمنشأة أخرى (التسلسل محدود بمستويين) |
| BRL-04 | عملية الحذف تتطلب تأكيداً صريحاً من المستخدم ولا يمكن التراجع عنها |
| BRL-05 | ربط الموظف بالمنشأة إلزامي — لا يمكن إضافة موظف بدون منشأة |

---

## 5. المتطلبات غير الوظيفية من منظور الأعمال

| الرقم | المتطلب |
|---|---|
| BR-NF-01 | يجب أن يكون النظام باللغة العربية بالكامل |
| BR-NF-02 | يجب أن يدعم النظام اتجاه RTL (من اليمين لليسار) في جميع العناصر |
| BR-NF-03 | يجب أن تكون الأرقام والعملات معروضة بالتنسيق السعودي |
| BR-NF-04 | يجب أن تكون بيانات المستخدم محفوظة بين الجلسات |

## 6. الوحدات المستقبلية (خارج النطاق الحالي)

- صلاحيات المستخدمين ومستويات الوصول
- إدارة الإجازات والغيابات
- رفع ومرفقات الوثائق (جوازات، هويات، عقود)
- تصدير البيانات إلى Excel/PDF
- نظام الإشعارات (انتهاء صلاحية الهوية، إلخ)
- الربط مع أنظمة الحكومة السعودية (مدد، مساند، إلخ)
