## 1. نظرة عامة على المشروع

### 1.1 الهدف من المشروع

بناء نظام ويب متكامل لإدارة الموارد البشرية يتيح للمؤسسات تسجيل وإدارة بيانات موظفيها ومنشآتها في مكان واحد، مع دعم كامل للغة العربية واتجاه النص من اليمين لليسار (RTL)، ومع إدارة متكاملة للمستخدمين والصلاحيات عبر API موثوق.

### 1.2 نطاق المشروع

يشمل النظام الوحدات التالية:
- **المصادقة والأمان** — تسجيل الدخول/الخروج، JWT، إدارة الجلسة
- **لوحة التحكم** — عرض إحصائيات وملخصات فورية
- **إدارة الموظفين** — تسجيل وتعديل وحذف بيانات الموظفين
- **إدارة المنشآت** — إدارة المنشآت الأساسية والفرعية
- **النقل والتحويل** — إدارة طلبات النقل الداخلي والخارجي
- **التقارير** — تقارير الرواتب وتوزيع الموظفين
- **مركز الخدمات** — بوابة موحدة لخدمات الموارد البشرية
- **إدارة المستخدمين** — إنشاء وتعديل وحذف حسابات المستخدمين ✦ جديد v2.0
- **إدارة الأدوار والصلاحيات** — تعريف أدوار مخصصة وتعيين صلاحيات دقيقة ✦ جديد v2.0
- **الملف الشخصي** — تحديث البيانات الشخصية وتغيير كلمة المرور ✦ جديد v2.0

## 3. متطلبات الأعمال

### 3.1 وحدة المصادقة والأمان ✦ جديد v2.0

| الرقم | المتطلب |
|---|---|
| BR-A-01 | يجب أن يتطلب النظام تسجيل الدخول بالبريد الإلكتروني وكلمة المرور قبل الوصول لأي وحدة |
| BR-A-02 | يجب أن يُصدر النظام رمز JWT صالح لمدة 24 ساعة عند تسجيل الدخول الناجح |
| BR-A-03 | يجب أن يُعيد النظام توجيه المستخدم إلى شاشة الدخول تلقائياً عند انتهاء صلاحية الرمز |
| BR-A-04 | يجب أن تحتوي بيانات المستخدم المُعادة من الخادم على قائمة الصلاحيات المُعيَّنة له |
| BR-A-05 | يجب أن تُخفي الواجهة عناصر القائمة والأزرار التي لا يملك المستخدم صلاحيتها |

### 3.2 وحدة لوحة التحكم

| الرقم | المتطلب |
|---|---|
| BR-D-01 | يجب أن يعرض النظام إجمالي عدد الموظفين المسجلين |
| BR-D-02 | يجب أن يعرض النظام عدد الموظفين النشطين |
| BR-D-03 | يجب أن يعرض النظام إجمالي عدد المنشآت |
| BR-D-04 | يجب أن يعرض النظام إجمالي كتلة الرواتب الشهرية بالريال السعودي |
| BR-D-05 | يجب أن يعرض النظام آخر طلبات النقل (حتى 5 طلبات) مع حالاتها |

### 3.3 وحدة إدارة الموظفين

| الرقم | المتطلب |
|---|---|
| BR-E-01 | يجب أن يتمكن المستخدم من إضافة موظف جديد بالبيانات الكاملة |
| BR-E-02 | يجب أن يكون الرقم التوظيفي فريداً لكل موظف |
| BR-E-03 | يجب أن يتمكن المستخدم من تعديل بيانات أي موظف |
| BR-E-04 | يجب أن يتمكن المستخدم من حذف موظف مع تأكيد مسبق |
| BR-E-05 | يجب أن يتمكن المستخدم من البحث عن موظف بالاسم أو الرمز أو الجنسية |
| BR-E-06 | يجب أن يتمكن المستخدم من تصفية الموظفين حسب الحالة الوظيفية |
| BR-E-07 | يجب أن تكون حالات الموظف: نشط، غير نشط، خروج نهائي، خروج مؤقت، اجازة |

**البيانات المطلوبة لكل موظف:**

*البيانات الشخصية:*
- الاسم الكامل (إلزامي)
- الرقم التوظيفي (إلزامي — فريد)
- الرقم القومي (إلزامي)
- رقم الهوية (إلزامي)
- الجنسية (إلزامي)
- نوع الموظف: سعودي / اجنبي (إلزامي)
- تاريخ انتهاء الهوية
- تاريخ دخول المملكة

*بيانات العمل:*
- المنشأة (إلزامي — اختيار من قائمة المنشآت المسجلة)
- المرتب الأساسي بالريال السعودي (إلزامي)
- حالة الموظف (إلزامي)
- المسمى الوظيفي
- الإدارة
- المدير المباشر
- موقع العمل
- الدرجة الوظيفية

*بيانات البنك والتواصل:*
- اسم البنك
- رقم الآيبان (IBAN)
- رقم الهاتف

## 4. قواعد العمل

| الرقم | القاعدة |
|---|---|
| BRL-01 | لا يمكن لموظفين اثنين امتلاك نفس الرقم التوظيفي |
| BRL-02 | لا يمكن إنشاء منشأة فرعية بدون تحديد منشأة أساسية |
| BRL-03 | لا يمكن تعيين منشأة فرعية كمرجع أب لمنشأة أخرى |
| BRL-04 | عملية الحذف تتطلب تأكيداً صريحاً ولا يمكن التراجع عنها |
| BRL-05 | ربط الموظف بالمنشأة إلزامي عند الإضافة |
| BRL-06 | لا يمكن حساب نسبة السعودة إذا كان إجمالي الموظفين = 0 |
| BRL-07 | لا يمكن إنشاء طلب نقل لموظف غير نشط |
| BRL-08 | رقم المرجع الحكومي إلزامي لإتمام أي نقل خارجي |
| BRL-09 | عند رفض/إلغاء طلب نقل خارجي تُعاد حالة الموظف إلى "نشط" تلقائياً |
| BRL-10 | لا يمكن حذف دور مرتبط بمستخدمين نشطين ✦ جديد |
| BRL-11 | البريد الإلكتروني فريد لكل حساب مستخدم — لا يقبل تكراراً ✦ جديد |
| BRL-12 | كلمة المرور يجب أن تحتوي على حرف كبير وحرف صغير ورقم، بحد أدنى 8 أحرف ✦ جديد |
| BRL-13 | يُسجَّل كل إجراء في سجل التدقيق (Audit Log) مع الطابع الزمني واسم المستخدم |
