## 1. نظرة عامة على المشروع

### 1.1 الهدف من المشروع

بناء نظام ويب متكامل لإدارة الموارد البشرية يتيح للمؤسسات تسجيل وإدارة بيانات موظفيها ومنشآتها في مكان واحد، مع دعم كامل للغة العربية واتجاه النص من اليمين لليسار (RTL).

### 1.2 نطاق المشروع

يشمل النظام الوحدات التالية:
- **لوحة التحكم** — عرض إحصائيات وملخصات فورية
- **إدارة الموظفين** — تسجيل وتعديل وحذف بيانات الموظفين
- **إدارة المنشآت** — إدارة المنشآت الأساسية والفرعية
- **التقارير** — تقارير الرواتب وتوزيع الموظفين
- **مركز الخدمات** — بوابة موحدة لخدمات الموارد البشرية
- **وحدة النقل والتحويل** — إدارة طلبات النقل الداخلي والخارجي مع التكامل مع المنصات الحكومية

## 2. أصحاب المصلحة

| الجهة | الدور |
|---|---|
| مدير النظام | المستخدم الرئيسي — يملك صلاحيات كاملة على جميع البيانات |
| إدارة الموارد البشرية | إدخال بيانات الموظفين، متابعة حالاتهم، إدارة طلبات النقل |
| الإدارة المالية | الاطلاع على تقارير الرواتب وتسويات النقل |

| الجهات الحكومية | منصة قوى وأبشر — تنفيذ الإجراءات القانونية لنقل الكفالة |

## 3. متطلبات الأعمال

### 3.1 وحدة لوحة التحكم

|---|---|

| BR-D-02 | يجب أن يعرض النظام عدد الموظفين النشطين |
| BR-D-03 | يجب أن يعرض النظام إجمالي عدد المنشآت |
| BR-D-04 | يجب أن يعرض النظام إجمالي كتلة الرواتب الشهرية بالريال السعودي |
| BR-D-05 | يجب أن يعرض النظام آخر الموظفين المضافين (حتى 6 موظفين) |
| BR-D-06 | يجب أن يعرض النظام توزيع الموظفين حسب المنشأة على شكل أشرطة بيانية |

### 3.2 وحدة إدارة الموظفين

| الرقم | المتطلب |
|---|---|
| BR-E-01 | يجب أن يتمكن المستخدم من إضافة موظف جديد بالبيانات الكاملة |
| BR-E-02 | يجب أن يكون الرقم التوظيفي فريداً لكل موظف |
| BR-E-03 | يجب أن يتمكن المستخدم من تعديل بيانات أي موظف |
| BR-E-04 | يجب أن يتمكن المستخدم من حذف موظف مع تأكيد مسبق |
| BR-E-05 | يجب أن يتمكن المستخدم من البحث عن موظف بالاسم أو الرقم التوظيفي أو الجنسية |
| BR-E-06 | يجب أن يتمكن المستخدم من تصفية الموظفين حسب المنشأة |
| BR-E-07 | يجب أن يتمكن المستخدم من تصفية الموظفين حسب الحالة الوظيفية |
| BR-E-08 | يجب أن تكون حالات الموظف: نشط، غير نشط، خروج نهائي، خروج مؤقت، اجازة |
| BR-E-09 | يجب أن يتمكن المستخدم من النقر على صف الموظف للانتقال إلى صفحة تفاصيل كاملة |
| BR-E-10 | يجب أن تعرض صفحة تفاصيل الموظف جميع بياناته مقسمةً في أقسام واضحة |
| BR-E-11 | يجب أن يتمكن المستخدم من إرفاق صورة شخصية للموظف |

**البيانات المطلوبة لكل موظف:**

*البيانات الشخصية:*
- الاسم الكامل (إلزامي)
- الرقم التوظيفي (إلزامي — فريد)
- الرقم القومي (إلزامي)
- رقم الهوية (إلزامي)
- الجنسية (إلزامي)
- نوع الموظف: سعودي / اجنبي (إلزامي)
- تاريخ انتهاء الهوية
- تاريخ دخول المملكة
- صورة شخصية (اختيارية)

*بيانات العمل:*
- المنشأة (إلزامي — اختيار من قائمة المنشآت المسجلة)
- المرتب الأساسي بالريال السعودي (إلزامي)
- حالة الموظف (إلزامي)
- عنوان العمل

*بيانات البنك والتواصل:*
- اسم البنك
- رقم الآيبان (IBAN)
- كود الدولة
- رقم الهاتف

| BR-F-07 | يجب أن تعرض صفحة التفاصيل: بيانات المنشأة، قائمة موظفيها، وقائمة المنشآت الفرعية (للأساسية فقط) |
| BR-F-08 | يجب أن يتمكن المستخدم من إضافة منشأة فرعية مباشرة من صفحة تفاصيل المنشأة الأساسية |
| BR-F-09 | يجب أن يتمكن المستخدم من تعديل بيانات المنشأة |
| BR-F-10 | يجب أن يتمكن المستخدم من حذف منشأة مع تأكيد مسبق |
| BR-F-11 | يجب أن تعرض صفحة تفاصيل المنشأة شجرة هرمية مرئية عند وجود علاقة أب/فرع — للمنشأة الأساسية: تُظهر نفسها كجذر ومنشآتها الفرعية كأفرع — للمنشأة الفرعية: تُظهر الأب كجذر وجميع الأخوات مع تمييز المنشأة الحالية — العقد في الشجرة قابلة للنقر للانتقال المباشر |

| تاريخ إصدار السجل التجاري | تاريخ إصدار السجل | لا |

| رقم المنشأة في التأمينات | رقم التسجيل في التأمينات الاجتماعية | لا |
| العنوان الوطني | رابط خريطة أو كود العنوان الوطني | لا |
| موقع العمل | وصف الموقع (حي، مدينة) | لا |

### 3.4 وحدة حساب نسبة السعودة (نطاقات)

| الرقم | المتطلب |
|---|---|
| BR-S-01 | يجب أن يعرض النظام نسبة التوطين الفعلية لكل منشأة تلقائياً عند فتح صفحة تفاصيلها |
| BR-S-02 | يجب أن يحسب النظام النسبة من عدد الموظفين السعوديين وغير السعوديين المرتبطين بالمنشأة |
| BR-S-03 | يجب أن يستعلم النظام جدول مرجعية Nitaqat لتحديد الحد الأدنى المطلوب حسب النشاط الاقتصادي |
| BR-S-04 | يجب أن يحدد النظام فئة المنشأة (بلاتيني / أخضر / أصفر / أحمر) بناءً على النسبة الفعلية والنسب المرجعية |
| BR-S-05 | يجب أن يعرض النظام حالة المنشأة (مقبول / غير مقبول) مقارنةً بالحد الأدنى للنشاط |
| BR-S-06 | يجب أن يعرض النظام رسالة مناسبة إذا لم يكن هناك موظفون أو لم يُحدَّد النشاط الاقتصادي |
| BR-S-07 | يجب أن يُحدِّث النظام حساب النسبة تلقائياً عند أي تغيير في بيانات الموظفين أو النشاط الاقتصادي |

**مدخلات الحساب:**
- عدد الموظفين بنوع `سعودي` المرتبطين بالمنشأة
- عدد الموظفين بنوع `اجنبي` المرتبطين بالمنشأة
- النشاط الاقتصادي للمنشأة (ISIC4 أو الاسم) → للبحث في جدول Nitaqat

**صيغة الحساب:**
```
النسبة الفعلية = (عدد السعوديين / إجمالي الموظفين) × 100
```

**فئات نطاقات:**

| الفئة | الشرط |
|---|---|
| أحمر (غير مقبول) | النسبة < الحد الأدنى للمنطقة الصفراء |
| أصفر (غير مقبول) | الحد الأدنى الأصفر ≤ النسبة < الحد الأدنى الأخضر |
| أخضر (مقبول) | الحد الأدنى الأخضر ≤ النسبة < العتبة البلاتينية |
| بلاتيني (مقبول) | النسبة ≥ العتبة البلاتينية |

### 3.7 وحدة النقل والتحويل

#### 3.7.1 النقل الداخلي

| الرقم | المتطلب |
|---|---|
| BR-T-01 | يجب أن يتمكن مدير الموارد البشرية من إنشاء طلب نقل داخلي لأي موظف نشط |
| BR-T-02 | يجب أن يدعم الطلب تغيير: الفرع، الإدارة، المسمى الوظيفي، المدير المباشر، موقع العمل، الراتب، الدرجة الوظيفية |
| BR-T-03 | يجب أن يدعم النظام تحديد تاريخ تفعيل مستقبلي للنقل |
| BR-T-04 | يجب أن يُرسَل الطلب تلقائياً للاعتماد عند الإنشاء |
| BR-T-05 | عند الاعتماد يجب أن يُحدِّث النظام ملف الموظف تلقائياً بجميع التغييرات |
| BR-T-06 | يجب أن يُسجِّل النظام جدول مقارنة (قبل/بعد) لجميع الحقول المعدَّلة |
| BR-T-07 | يجب أن يُضاف سجل النقل في تاريخ الموظف (History) |

#### 3.7.2 النقل الخارجي — خروج

| الرقم | المتطلب |
|---|---|
| BR-T-08 | يجب أن يدعم النظام نوعين: نقل كفالة وإعارة مؤقتة |
| BR-T-09 | يجب تسجيل: اسم الجهة المستقبِلة، سبب النقل، التاريخ المتوقع |
| BR-T-10 | يجب أن يحسب النظام تسوية نهائية تقديرية (راتب معلق + رصيد إجازة − قروض) |
| BR-T-11 | عند الاعتماد يجب تجميد ملف الموظف وتغيير حالته إلى "خروج مؤقت" |
| BR-T-12 | يجب أن يتوقف النظام عند نقطة الإجراء الحكومي ويعرض تعليمات واضحة |
| BR-T-13 | يجب أن يوفر النظام روابط مباشرة لمنصة قوى (qiwa.sa) وأبشر (absher.sa) |
| BR-T-14 | بعد إتمام الإجراء الحكومي يجب أن يتمكن المستخدم من إدخال رقم المرجع الحكومي |
| BR-T-15 | عند التأكيد النهائي تُحدَّث حالة الموظف إلى "خروج نهائي" ويُغلق سجله |

#### 3.7.3 النقل الخارجي — دخول

| الرقم | المتطلب |
|---|---|
| BR-T-16 | يجب أن يدعم النظام ربط موظف محوَّل من منشأة أخرى |
| BR-T-17 | يجب تسجيل اسم الجهة المُرسِلة والتاريخ المتوقع |
| BR-T-18 | بعد إتمام الإجراء الحكومي وإدخال رقم المرجع تُفعَّل حالة الموظف تلقائياً |

#### 3.7.4 سير العمل والحالات

**مسار النقل الداخلي:**
```
مسودة → قيد الاعتماد → مكتمل (تحديث ملف الموظف فوري)
```

**مسار النقل الخارجي:**
```
مسودة → قيد الاعتماد → معتمد (تجميد الملف) → معلق – إجراء حكومي → مكتمل
```

**حالات الطلب:**

| الحالة | الوصف |
|---|---|
| `draft` | مسودة — لم يُرسَل بعد |
| `pending_approval` | قيد الاعتماد — في انتظار القرار |
| `approved` | معتمد — جارٍ تجهيز الإجراءات |
| `pending_government` | معلق — في انتظار إجراء حكومي خارجي |
| `completed` | مكتمل — أُتمَّت جميع الخطوات |
| `rejected` | مرفوض — مع حفظ سبب الرفض |
| `cancelled` | ملغى — تم إلغاؤه من قِبل المستخدم |

## 4. قواعد العمل

| الرقم | القاعدة |
|---|---|
| BRL-01 | لا يمكن لموظفين اثنين امتلاك نفس الرقم التوظيفي |
| BRL-02 | لا يمكن إنشاء منشأة فرعية بدون تحديد منشأة أساسية |
| BRL-03 | لا يمكن تعيين منشأة فرعية كمرجع أب لمنشأة أخرى (التسلسل محدود بمستويين) |
| BRL-04 | عملية الحذف تتطلب تأكيداً صريحاً من المستخدم ولا يمكن التراجع عنها |
| BRL-05 | ربط الموظف بالمنشأة إلزامي — لا يمكن إضافة موظف بدون منشأة |
| BRL-06 | لا يمكن حساب نسبة السعودة إذا كان إجمالي الموظفين = 0 |
| BRL-07 | لا يمكن تحديد فئة نطاقات إذا لم يُعيَّن النشاط الاقتصادي للمنشأة |
| BRL-08 | الحد الأدنى المطلوب (الفئة الخضراء) يُحدَّد بناءً على قسم ISIC4 للنشاط |
| BRL-09 | لا يمكن إنشاء طلب نقل داخلي لموظف غير نشط |
| BRL-10 | لا يمكن إنشاء طلب نقل خروج لموظف غير نشط |
| BRL-11 | النظام لا يُنفِّذ الإجراءات القانونية لنقل الكفالة — يتوقف عند نقطة التسليم الحكومي ويُرشد المستخدم فقط |
| BRL-12 | رقم المرجع الحكومي إلزامي لإتمام أي نقل خارجي — لا يمكن تأكيد الإتمام بدونه |
| BRL-13 | عند رفض أو إلغاء طلب نقل خارجي كانت قد غيَّرت حالة الموظف، تُعاد حالته إلى "نشط" تلقائياً |
| BRL-14 | يُسجَّل كل إجراء في سجل التدقيق (Audit Log) مع الطابع الزمني واسم المستخدم |
